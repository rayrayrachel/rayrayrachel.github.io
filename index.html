[PROGRAMMING] Visual Computing

[Solo Project] Developing a Photoshop application || Java Intellij || February 2024

Developing Photoshop

I developed an application that can apply gamma correction, nearest neighbor interpolation, bilinear interpolation and cross correlation to an image, pixel by pixel. The filter used is laplacian 5x5 filter.

IMAGE LOOP
- Image are 2D arrays of RGB vlaues
- Pixel position = I[Xcoordinate][Ycoordinate]
- Color of the pixel = I[Xcoordinate][Ycoordinate][color]

GAMMA CORRECTION
- Aim: to adjust the intensity(brightness) of the image
- Euqation: newIntensity = (originalIntensity) ^ ( 1 / gammaValue)
- Java normalisation: 0..255 value => 0..1 in java
- Gamma lookup table: match value with the table to reduce computing time

IMAGE RESIZEING - Nearest Neighbour Interpolation
- Aim: to reduce/ expand the image using nearest neighbour interpolation
- Equation: original_x_coordinate = new_x_coordinate * (original_width / new_width)
- Equation: original_y_coordinate = new_y_coordinate * (original_height / new_height)
- Finally convert them to the nearest integer

INAGE RESIZING - Bilinear Interpolation
- Aim: To reduce/expand the image using bilinear interpolation.
- Get the four corners' color.
- Apply linear interpolation horizontally.
- Apply linear interpolation vertically.
- Apply linear interpolation to the above two values.
Linear interpolation equation for horizontal interpolation:
new_color_value = original_color_value_at_x1 + (original_color_value_at_x2 - original_color_value_at_x1) * ((new_x_coordinate - original_x1_coordinate) / (new_x2_coordinate - original_x1_coordinate))
Linear interpolation equation for vertical interpolation:
new_color_value = original_color_value_at_y1 + (original_color_value_at_y2 - original_color_value_at_y1) * ((new_y_coordinate - original_y1_coordinate) / (new_y2_coordinate - original_y1_coordinate))

CROSS CORRELATION - LAPLACIAN FILTER
- Aim: apply laplacian filter to the image
- Find the weighted sums of each pixel in the three different color channels by multiplying the pixel values and the corresponding 5x5 pixel values with the filter.
- Utilize a loop to find the maximum and minimum values of the three color channels.
- Create the final image by normalizing the values using the formula:
normalizedValue = (weightedSums - min) / (max - min).

[MACHINE LEARNING] 8 Queens Puzzle - Genetic Algorithms

[Solo Project] Solve the 8 queen Puzzle with Genetic Algorithms || Python || February 2024

Genetic Algorithm

I applied genetic algorithm to solve the 8-queen-puzzle. The aim is to find the location of 8 queens on a chessboard such that they cannot attack each other.

I have functions to randomly generated the list of coordinates of the seven queens in different columns. The fixed queen's coordinates were inserted based on my student ID’s modulo plus 1, with coordinates ranging from 1 to 8, originating at the bottom left.

Functions are implemented to flatten the list and extract the x-coordinates. Other functions calculate the number of pairs of attacking queens: horizontally, vertically, and diagonally, as well as determine the fitness level (maximum conflict(28) minus the board conflicts).

Applying genetic algorithm, the population and generation size is set to 150 and 200. For each generation, there are 150 randomly generated queen coordinate with the fixed queen. Individual fitness is then calculated, and the orientations are sorted in descending order. If the first item in the list equals 28 (maximum fitness), the result is returned, indicating that the puzzle is solved. If there is no solution in this generation, the program selects the first half of the population list (highest fitness) as parents. It performs splitting at random indexes (unchanged fixed queen), resulting in swapping along the split to create offspring. Afterward, mutations (random swapping of two queens' x-coordinates) are applied, resulting in a final population of 75 parents and 75 offspring. This process is repeated 200 generations until result is found.

To find all solutions, the function is looped 100 times, appending and returning successful solutions. The sampling size is set to be sufficiently large for capturing all solutions efficiently.

Collect all the chips scattered across the landscape.

[PROGRAMMING] Digit Invaders

[Team Project] Developing a classic digital game || C++ C# GUI || March 2021

PolyU Invaders

In a group of two, we revamped the classic CASIO calculator game 'Digit Invader,' transforming it into a cool graphical computer game.

Backstory:
The story began when people implemented this game into a CASIO calculator. Students used to be addicted to this simple calculator game.

Game Instructions:
First, this game will generate digits from zero to nine. Subsequently, when players shoot a digit which can be divided by 10 without remainders, a monster (n) will be generated. Players need to press “FIRE” to cancel the digits or monsters (n) displayed on the screen by pressing “AIM” to change the digit on the left.

Initially, there are 9 underscores. When all underscores turn into digits, players lose one round and lose a try (life). Players have three tries (lives) to score marks. One mark is earned when players cancel one digit. Five marks are earned when players cancel one monster (n). Players win when they score 100; the maximum score of the game is, therefore, 100.

Functions:
This digital game can log into different user, store scores into user profiles, has highs-core table, admin configuration and the gameplay.
